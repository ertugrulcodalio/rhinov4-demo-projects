# Implementation Plan: Blog Feature

## Summary
Add a Blog model owned by an organization. Admins can manage blogs, and users can view them.

## PRD gaps compensated
None — PRD sections are aligned.

## Blueprint inventory
A blueprint YAML has been written for each of these models. Develop stage will run
`bundle exec rails agentcode:blueprint` to materialize them.

- `.agentcode/blueprints/_roles.yaml` — roles: admin, user
- `.agentcode/blueprints/blogs.yaml` — `Blog`

## Models (reference)

### `Blog`
- Ownership: organization-scoped
- Key fields: title, body, published
- Relationships: belongs_to Organization

## Extras (NOT covered by blueprint — develop stage implements these by hand)
None — blueprint covers every requirement.

## Screens
> **Rhino auto-generates index / show / create / update / destroy screens** for every model registered in `config.resources`.

## Implementation order (recommended stage flow)

1. **Plan** — this artifact set (YAMLs, plan, criteria, index).
2. **Develop** — phase A: `bundle exec rails agentcode:blueprint` + migrations. Phase B: implement **Extras**. Phase D: write backend unit + integration tests for every AC.
3. **UI / UI-Integration** — build screens listed above.
4. **Test** — run `bundle exec rails test` to confirm green and tighten anything broken.
5. **Review** — PRD compliance + test/coverage report.
